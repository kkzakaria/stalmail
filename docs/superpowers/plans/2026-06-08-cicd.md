# Stalmail CI/CD — Plan d'implémentation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mettre en place 3 workflows GitHub Actions (CI, release-please, CD) et la config release-please pour automatiser tests, versioning et publication de l'image Docker multi-plateforme.

**Architecture:** CI tourne sur chaque push et PR. release-please analyse les commits Conventional Commits et gère les Release PRs + tags. CD publie l'image Docker `linux/amd64,linux/arm64` sur ghcr.io au push sur `main` (`latest`) et sur les tags versionnés créés par release-please.

**Tech Stack:** GitHub Actions, release-please v4, docker/build-push-action v6, docker/metadata-action v5, oven-sh/setup-bun v2, ghcr.io

### Task 3 : Workflow CI

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Créer le dossier workflows**

```bash
mkdir -p .github/workflows
```

- [ ] **Créer `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    name: Lint, Typecheck & Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Cache Bun dependencies
        uses: actions/cache@v4
        with:
          path: ~/.bun/install/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lock') }}
          restore-keys: |
            ${{ runner.os }}-bun-

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Lint
        run: bun run lint

      - name: Typecheck
        run: bun run typecheck

      - name: Test
        run: bun run test
```

### Task 5 : Workflow CD

**Files:**
- Create: `.github/workflows/cd.yml`

- [ ] **Créer `.github/workflows/cd.yml`**

```yaml
name: CD

on:
  push:
    branches: [main]
    tags: ['v*.*.*']

permissions:
  packages: write
  contents: read

jobs:
  build-and-push:
    name: Build & Push Docker image
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract Docker metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository }}
          tags: |
            type=raw,value=latest,enable={{is_default_branch}}
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=semver,pattern={{major}}

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          platforms: linux/amd64,linux/arm64
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```
